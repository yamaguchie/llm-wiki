# パイプラインワークフロー仕様

## 全体の流れ

```
Step 1: データクレンジング
│ 1.1. RAWデータ — PDFアップロード→抽出→チャンク→埋め込み（1.2+2.1自動クリア）
│ 1.2. LLM-Wiki — RAWチャンクからLLMがエンティティページ生成（2.1自動クリア）
│
Step 2: ドメイン知識の付与
│ 2.1. QA（質問＋回答ペア） — LLM生成＋3択レビュー＋新規追加（進捗バー付き非同期）
│ 2.2. オントロジー定義 — QA＋LLM-WikiからLLMが生成（進捗バー付き非同期）
│ 2.3. オントロジー図 — 定義からSVGフォースグラフで可視化
│
Step 3: ナレッジグラフ
│ 3.1. KG構築 — LLM-Wikiから抽出→Neo4j投入（進捗バー付き非同期）
│ 3.2. QAのAgent回帰テスト — 全QA検証＋原因解析＋失敗QAから修正
```

## 各Stepの詳細

### 1.1 RAWデータ

- **入力**: 業務マニュアルPDF（ドラッグ&ドロップ / ファイル選択）
- **処理**: pdfplumber抽出 → 800文字チャンク分割 → Gemini埋め込み
- **出力**: `step1_data/raw/chunks.json` + `step1_data/raw/raw_embeddings.json`
- **進捗**: プログレスバー（PDF保存→抽出→チャンク→埋め込み→クリーンアップ）
- **後処理**: 抽出後、チャンク一覧表示（クリックで全文モーダル）
- **自動クリア**: 1.2 LLM-Wiki + 2.1 QA + 3.2 検証結果

### 1.2 LLM-Wiki

- **入力**: `step1_data/raw/chunks.json`（最大100チャンク）
- **処理**: 10バッチずつLLMに投入 → 1概念1ファイルのMarkdown生成
- **形式**: 標準Markdownリンク（`[text](page.md)`）、YAML frontmatter（`page: N`）
- **出力**: `step1_data/wiki/*.md`（62ページ）
- **進捗**: プログレスバー
- **自動クリア**: 2.1 QA + 3.2 検証結果

### 2.1 QA（質問＋回答ペア）

- **入力**: `step1_data/wiki/*.md`
- **生成**: LLMが質問＋期待回答＋タイプ＋トレース（参照経路）をJSONで生成
- **タイプ**: 📖単一参照 / 🔗多段探索 / 📋一覧取得 / ⚠️条件確認
- **レビュー**: 「正しい」「修正必要」「誤り」の3択 + コメント
- **新規追加**: 質問 + 説明 + 期待回答 + タイプ選択
- **一括操作**: 全QA承認 / 全QA削除
- **進捗**: プログレスバー付き非同期実行

### 2.2 オントロジー定義

- **入力**: 承認済みQA + `step1_data/wiki/*.md`
- **方式**:
  - ① Wikiから定義: LLM-Wikiから直接クラス/関係を抽出
  - ②③ QAで反復修正: 承認済みQAの充足度を評価しながら反復修正
  - 一括生成: ①+②③ をまとめて実行
- **出力**: クラス定義（名前・論理名・プロパティ）+ 関係定義 + 制約（クラス.プロパティに紐付け）
- **永続化**: `step3_kg/neo4j/generated_constraints.cypher` + `agent/active_constraints.json`
- **進捗**: プログレスバー付き非同期実行

### 2.3 オントロジー図

- **入力**: 2.2 オントロジー定義
- **表示**: SVGフォースグラフ（定義済みクラス=青、参照クラス=灰）
- **操作**: ノードクリックでプロパティ表示

### 3.1 ナレッジグラフ構築

- **入力**: 2.2 オントロジー定義 + `step1_data/wiki/*.md`
- **処理**: LLM-Wikiから実体抽出 → Neo4j投入 / Cypher出力
- **ビュー**: Neo4j実データ / 抽出JSON 切替表示
- **SVGグラフ**: ノードクリックで全プロパティ＋入出力エッジ表示
- **進捗**: プログレスバー付き非同期実行

### 3.2 QAのAgent回帰テスト

- **入力**: 承認済みQA
- **3方式比較**: ナイーブRAG / KGのみ / KG+Wiki補完
- **サマリー**: 正解率 + 正解+部分率（パーセント表示）
- **原因解析**: ノード欠落 / プロパティ不足 / 値誤り
- **修正**: 「🔧 失敗QAからオントロジー修正」→ 原因解析結果をLLMに渡して再生成

## 一括実行

チャット画面の「▶ 全行程実行」ボタンで 1.2〜3.2 を自動実行:

| ステップ | 進捗 | 内容 |
|---|---|---|
| Step 1/6 | 0-15% | LLM-Wiki 生成 |
| Step 2/6 | 15-30% | QA 生成 |
| Step 3/6 | 30-35% | QA 一括承認 |
| Step 4/6 | 35-65% | オントロジー定義 生成 |
| Step 5/6 | 65-85% | KG 抽出 + Neo4j 投入 |
| Step 6/6 | 85-100% | QA 検証 |

## 連鎖的クリーンアップ

| 操作 | 自動削除されるデータ |
|---|---|
| 1.1 RAWデータ再作成 | 1.2 LLM-Wiki + 2.1 QA + 3.2 検証結果 |
| 1.2 LLM-Wiki再生成 | 2.1 QA + 3.2 検証結果 |
